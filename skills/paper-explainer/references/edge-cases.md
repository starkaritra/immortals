# Edge Cases

## Math-Heavy Paper (e.g., pure theory)
Spend more time on equations. Build up from simpler cases. Use diagrams to show geometric or graphical intuitions. Put formal proofs in Deep Dive blocks.

## Empirical / ML Paper
Focus on architecture diagrams, training procedure, and result tables. Include comparison table vs prior methods.

## Short Paper / Workshop Paper
Still do full treatment but more concise. Skip Deep Dives if they would dwarf the paper itself.

## Paper with No Accessible Full Text (Paywall)
Work from abstract + any visible content + known context about the paper. Flag clearly what is inferred vs. stated. Offer the user tips to access it (e.g., Unpaywall, Semantic Scholar, author's homepage, arXiv preprint).

## Very Long Paper (50+ pages)
After the Big Picture block (Step 1), ask the user if they want full coverage or specific sections. Always complete background + knowledge graph regardless.

## Survey / Review Paper
Replace section-by-section walkthrough with a taxonomy diagram and comparison table. Explain each major family of methods rather than each section.

## Systems / Hardware Paper
Add extra background on the hardware architecture. Include a comparison table of prior system benchmarks. Use Deep Dives for micro-architectural details.

## Multi-Paper / Reading List
If the user pastes multiple URLs, process one at a time and ask which to start with, unless they specify an order.
