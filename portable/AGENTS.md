# AS agent suite — Codex / opencode / Cursor / Amp adapter

> Generated by `scripts/gen_adapters.py` from `registry/*.json` + `skills/INDEX.json`. Do not edit by hand — edit the source and regenerate.

This project runs the **AS agent suite**: a set of specialist agents that call **skills** (self-contained capability packs) on demand. Skills are loaded lazily to keep context small.

## Using skills (lazy loading)

Skills are served by the **`as-skills` MCP server**. Do not preload them — load only what a task needs:

1. **Scan the directory** — `skill_list(agent?)` returns `{name, description, owner_agent}` for each skill (optionally scoped to one agent). This is the cheap always-on index.
2. **Load on demand** — when a skill's `description` matches the task, call `skill_get(name)` to pull its full `SKILL.md` into context.
3. **Deep detail** — call `skill_get_reference(name, path)` only for a specific `references/` or `assets/` file when that phase needs it.

Register the servers in your host's MCP config (Codex: `portable/mcp/codex.config.toml`; Cursor: `portable/mcp/claude.mcp.json`).

## Typed contracts (quality floor)

Every seam is a JSON-Schema contract in `immortals/contracts/schemas/` (`registry/v1`, `plan/v1`, `artifact/v1`, `event/v1`). Validate artifacts against them regardless of the underlying model — this is what keeps output correct when the host or model changes.
